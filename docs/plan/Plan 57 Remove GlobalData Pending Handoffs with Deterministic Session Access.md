## Task Analysis
- Main objective: Remove transient `app.globalData` session handoff channels (`pendingPersistedMatchState`, `pendingHomeMatchState`, `sessionHandoff`) and move all handoff-dependent flows to deterministic session access through the unified active-session storage service, with atomic update helpers for safe read-modify-write operations.
- Identified dependencies: Task 53 canonical session contract in `utils/match-state-schema.js`; Task 54 unified storage service in `utils/active-session-storage.js`; existing handoff usage in `page/game.js`, `page/index.js`, `page/setup.js`, `utils/active-session-storage.js`, `utils/app-data-clear.js`, `app.js`; test harnesses with source rewriting in `tests/game-screen-layout.test.js`, `tests/setup-flow.test.js`, `tests/home-screen.test.js`, `tests/summary-screen.test.js`, plus storage tests in `tests/active-session-storage.test.js` and `tests/active-session-storage.integration.test.js`.
- System impact: High on session continuity during setup->game->home navigation and app lifecycle save paths; medium on persistence migration behavior and integration test fixtures; medium-high concurrency risk if atomic updates are added without clear locking/reentrancy behavior.

## Chosen Approach
- Proposed solution: Use a service-first, bounded refactor in five phases mapped to subtasks 57.1-57.5: (1) audit handoff producers/consumers, (2) implement atomic session update helpers in unified storage service, (3) refactor page-level handoff reads/writes to deterministic service access, (4) tighten module interfaces to avoid implicit handoff dependencies, and (5) remove handoff-specific globalData code and finalize concurrency protections.
- Justification for simplicity: Deepthink comparison selected the smallest approach that still resolves race risk. Option A (page-only replacement) is fast but duplicates update logic and leaves race windows; Option B (chosen: add minimal service-level atomic helpers + targeted page cleanup) centralizes correctness with limited churn; Option C (full runtime state architecture rewrite removing all `globalData.matchState` usage) is overengineered for this task and increases regression risk.
- Components to be modified/created: Modify `utils/active-session-storage.js` (new atomic helpers, optional internal lock/reentrancy guard, remove handoff migration branches), `page/game.js`, `page/index.js`, `page/setup.js`, `app.js`, `utils/app-data-clear.js`; potentially delete or deprecate `utils/session-handoff.js` if unused; update tests in `tests/active-session-storage.test.js`, `tests/active-session-storage.integration.test.js`, `tests/game-screen-layout.test.js`, `tests/setup-flow.test.js`, `tests/home-screen.test.js`, `tests/summary-screen.test.js`; add/refresh task development log entry for audit evidence.

## Implementation Steps
1. Phase 57.1 - Audit and discovery baseline: run static discovery and produce a handoff map (writer -> key -> reader -> flow) before editing. Use:
   - `rg -n "pendingPersistedMatchState|pendingHomeMatchState|sessionHandoff|consumeSessionHandoff|consumeHomeHandoff|storeSessionHandoff|storeResumeSessionHandoff|storeHomeHandoff" app.js page utils tests`
   - `rg -n "globalData\\.(pending|sessionHandoff)" app.js page utils tests`
   - `rg -n "migrateLegacySessions\(|pendingRuntimeMatchState|pendingPersistedMatchState" utils/active-session-storage.js tests`
   Record exact files/functions and classify each usage as legacy migration, runtime handoff, or unrelated global runtime state.
2. Phase 57.2 - Implement atomic helper APIs in unified storage service: add `updateActiveSession(updaterFn, options?)` and `updateActiveSessionPartial(patch, options?)` in `utils/active-session-storage.js`, built on `getActiveSession()` + `saveActiveSession()` with explicit null handling and immutability guarantees (never mutate caller object). Include a small in-process critical-section guard (single-writer lock/reentrancy protection) to serialize update helper execution.
3. Phase 57.2 - Add focused unit coverage for atomic helpers: extend `tests/active-session-storage.test.js` for (a) null-session no-op behavior, (b) immutable update behavior, (c) updater exception handling, (d) sequential update ordering, and (e) preservation of canonical contract fields (`schemaVersion`, `updatedAt` semantics).
4. Phase 57.3 - Remove transient handoff writes: refactor `page/setup.js`, `page/game.js`, and `page/index.js` to stop writing `globalData.pendingPersistedMatchState` and `globalData.pendingHomeMatchState`; replace those transitions with deterministic save/read through unified service (`saveActiveSession`, `getActiveSession`, and helper updates where read-modify-write is needed).
5. Phase 57.3 - Remove transient handoff reads: refactor validation/resume logic (`page/game.js` session access guard, `page/index.js` resume visibility and resume click revalidation) to rely only on deterministic service reads with explicit null/invalid-session handling and existing navigation fallbacks.
6. Phase 57.4 - Update module interfaces for direct session access: remove methods that only exist for handoffs (`consumeSessionHandoff`, `consumeHomeHandoff`, `storeSessionHandoff`, `storeResumeSessionHandoff`, `storeHomeHandoff`) and adjust callers to pass explicit state or fetch from unified storage directly; keep `globalData.matchState` runtime usage untouched unless directly tied to pending handoff behavior.
7. Phase 57.5 - Cleanup + concurrency hardening: remove handoff-specific branches/options in `migrateLegacySessions` and related cleanup code paths (`utils/active-session-storage.js`, `app.js`, `utils/app-data-clear.js`), then remove dead compatibility artifacts/comments/constants (including `utils/session-handoff.js` if confirmed unused).
8. Elevated risk control (concurrency/lifecycle): in app/page lifecycle paths where both `page.onDestroy` and `app.onDestroy` can persist near-simultaneously, ensure write behavior is deterministic (single service path + update helper lock). Mitigation: keep last-write semantics based on canonical session timestamps and avoid mixed runtime/handoff fallbacks. Rollback note: if lifecycle regression appears, temporarily restore previous fallback behavior behind one isolated revert commit while keeping atomic helper tests intact.
9. Test harness alignment for refactored imports and behavior: update source-rewrite test utilities to match any import changes (for example from `match-storage` to `active-session-storage`) and remove expectations tied to pending handoff keys; refresh integration fixtures that previously asserted handoff migration behavior.
10. Final verification + review handoff package: execute targeted tests, full QA command, and static no-handoff scans; attach audit matrix and risk outcomes to task log/PR for reviewer traceability.
